## What is Diffusion?

Here is the core idea in one sentence: **if you can learn to reverse gradual noise corruption, you can generate data from pure noise.**

Think of it like this. Imagine you take a photo and add a tiny bit of static. Then a bit more. Then more, and more, until the image is nothing but TV snow. That is the *forward process* — easy, deterministic, requires no learning.

Now imagine you train a neural network to look at a noisy image and predict what it looked like one step *less* noisy. If that network is good enough, you can start from pure static and iteratively clean it up until a brand-new image appears. That is the *reverse process* — and that is what a diffusion model learns.

This idea was formalized in the 2020 paper **"Denoising Diffusion Probabilistic Models"** (DDPM) by Ho, Jain, and Abbasi-Srinivasan. It built on earlier work by Sohl-Dickstein et al. (2015) but made the training simple and the results stunning.

**How does it compare to other generative models?**

- **GANs** (Generative Adversarial Networks) train two networks against each other — a generator and a discriminator. They produce sharp results but are notoriously finicky to train (mode collapse, training instability).
- **VAEs** (Variational Autoencoders) learn to encode data into a latent space and decode it back. They are stable to train but tend to produce blurry outputs.
- **Diffusion models** sidestep both problems. Training is a straightforward regression loss (predict the noise), and the iterative sampling produces high-quality results. The tradeoff is speed — sampling requires many steps. But the simplicity is remarkable.

We are going to build one from scratch, and we are going to do it in 2D so you can *see* every step.

## The Forward Process

The forward process gradually destroys the data by adding noise over T timesteps. At each step t, we add a bit more Gaussian noise according to a schedule. By the final step T, the data is indistinguishable from pure random noise.

The beautiful part: we do not actually have to run through all T steps sequentially. There is a closed-form formula that lets us jump straight to any timestep t.

**The noise schedule**

We define a sequence of values called `beta_t` — one for each timestep. These control how much noise to add at each step. In the original DDPM paper, beta goes linearly from 0.0001 to 0.02 over T=1000 steps.

<CodeBlock title="schedule.py" language="python">
{"T = 1000  # total number of timesteps\nbeta = torch.linspace(0.0001, 0.02, T)  # noise schedule"}
</CodeBlock>

From beta we derive two more sequences:

- `alpha_t = 1 - beta_t` — the fraction of signal retained at each step.
- `alpha_bar_t = product of all alpha values from step 1 to step t` — the *cumulative* signal retention.

<CodeBlock title="schedule.py" language="python">
{"alpha = 1.0 - beta\nalpha_bar = torch.cumprod(alpha, dim=0)  # cumulative product"}
</CodeBlock>

`torch.cumprod` is the cumulative product — like `Array.reduce` but it keeps every intermediate result. So `alpha_bar[t]` equals `alpha[0] * alpha[1] * ... * alpha[t]`.

At the start (t=0), `alpha_bar` is close to 1 — almost all signal is preserved. By the end (t=999), `alpha_bar` is close to 0 — almost all signal is gone.

**The key equation**

Here is the magic formula. To get a noisy version of `x_0` at any timestep `t`:

```
q(x_t | x_0) = N(sqrt(alpha_bar_t) * x_0, (1 - alpha_bar_t) * I)
```

In plain English: `x_t` is a Gaussian distribution centered at a scaled-down version of `x_0`, with variance that grows as `t` increases. When `alpha_bar_t` is close to 1 (early steps), the center is close to `x_0` and the variance is small — barely noisy. When `alpha_bar_t` is close to 0 (late steps), the center is near zero and the variance is near 1 — pure noise.

In code, the sampling trick turns this into:

```
x_t = sqrt(alpha_bar_t) * x_0 + sqrt(1 - alpha_bar_t) * epsilon
```

where `epsilon` is standard Gaussian noise (`torch.randn_like(x_0)`). The first term shrinks the original data, the second term adds noise. That is it.

<CodeBlock title="forward.py" language="python">
{"def forward_diffusion(x_0, t, alpha_bar):\n    \"\"\"Add noise to x_0 at timestep t.\n    \n    Args:\n        x_0: original data, shape (batch, 2)\n        t: timestep indices, shape (batch,)\n        alpha_bar: cumulative alpha schedule, shape (T,)\n    \n    Returns:\n        x_t: noisy data, shape (batch, 2)\n        noise: the noise that was added, shape (batch, 2)\n    \"\"\"\n    # Gather the alpha_bar values for each sample's timestep\n    a_bar = alpha_bar[t].unsqueeze(1)  # shape: (batch, 1)\n    \n    noise = torch.randn_like(x_0)       # sample epsilon ~ N(0, I)\n    x_t = torch.sqrt(a_bar) * x_0 + torch.sqrt(1 - a_bar) * noise\n    return x_t, noise"}
</CodeBlock>

A few things to note:

- `alpha_bar[t]` uses the timestep indices to look up the right alpha_bar value for each sample. This is fancy indexing — like `t.map(i => alpha_bar[i])` in JS.
- `.unsqueeze(1)` adds a dimension so the shape goes from `(batch,)` to `(batch, 1)`. This is needed so the multiplication broadcasts correctly across the 2 coordinates.
- `torch.randn_like(x_0)` generates noise with the same shape as `x_0`. It is standard normal — mean 0, variance 1.

Let us visualize what happens to our spiral at different noise levels:

<CodeBlock title="visualize_forward.py" language="python">

- `torch.full((len(dataset),), step, dtype=torch.long)` creates a tensor filled with the same timestep value. It is like `new Array(2000).fill(step)`.

## The Noise Predictor

Now we need a neural network that can look at noisy data `x_t` and predict the noise `epsilon` that was added. This is the heart of the diffusion model.

The network needs two inputs:
1. `x_t` — the noisy 2D point (2 values)
2. `t` — the timestep (tells the network how noisy the input is)

**Time conditioning with sinusoidal embeddings**

We cannot just feed the raw timestep integer into the network — a single number does not give the network much to work with. Instead, we use sinusoidal position embeddings (the same trick from the Transformer paper) to convert the timestep into a rich vector.

<CodeBlock title="time_embed.py" language="python">
{"def sinusoidal_embedding(t, embed_dim=64):\n    \"\"\"Convert integer timesteps to sinusoidal embeddings.\n    \n    Args:\n        t: timestep indices, shape (batch,)\n        embed_dim: dimension of the embedding\n    \n    Returns:\n        embedding, shape (batch, embed_dim)\n    \"\"\"\n    half = embed_dim // 2\n    # Frequencies: exp(-log(10000) * i / half) for i in 0..half-1\n    freqs = torch.exp(\n        -np.log(10000.0) * torch.arange(half, dtype=torch.float32) / half\n    ).to(t.device)\n    \n    # Outer product: each timestep times each frequency\n    args = t.float().unsqueeze(1) * freqs.unsqueeze(0)  # (batch, half)\n    \n    # Interleave sin and cos\n    return torch.cat([torch.sin(args), torch.cos(args)], dim=1)  # (batch, embed_dim)"}
</CodeBlock>

This produces a 64-dimensional vector for each timestep. Different timesteps map to very different vectors, which helps the network distinguish "barely noisy" from "extremely noisy."

If you have seen positional encodings in Transformers, this is the exact same idea — different frequencies encode different scales of position.

**The MLP architecture**

Our noise predictor is a simple feedforward network (MLP — Multi-Layer Perceptron). We concatenate the 2D point with the 64D time embedding to get a 66D input, then pass it through three linear layers with ReLU activations.

<CodeBlock title="model.py" language="python">
{"class NoisePredictor(nn.Module):\n    \"\"\"MLP that predicts the noise added to a 2D point.\"\"\"\n    \n    def __init__(self, time_embed_dim=64, hidden_dim=128):\n        super().__init__()\n        self.time_embed_dim = time_embed_dim\n        \n        # Input: 2 (point coords) + 64 (time embedding) = 66\n        self.net = nn.Sequential(\n            nn.Linear(2 + time_embed_dim, hidden_dim),\n            nn.ReLU(),\n            nn.Linear(hidden_dim, hidden_dim),\n            nn.ReLU(),\n            nn.Linear(hidden_dim, 2),  # output: predicted noise (2D)\n        )\n    \n    def forward(self, x_t, t):\n        \"\"\"Predict the noise in x_t.\n        \n        Args:\n            x_t: noisy points, shape (batch, 2)\n            t: timestep indices, shape (batch,)\n        \n        Returns:\n            predicted noise, shape (batch, 2)\n        \"\"\"\n        t_emb = sinusoidal_embedding(t, self.time_embed_dim)  # (batch, 64)\n        inp = torch.cat([x_t, t_emb], dim=1)                  # (batch, 66)\n        return self.net(inp)                                    # (batch, 2)"}
</CodeBlock>

Let us unpack the PyTorch patterns here:

- `nn.Module` is the base class for all neural network components in PyTorch. Think of it like React's `Component` class — you extend it and define your structure.
- `__init__` is the constructor. You define your layers here.
- `nn.Sequential` chains layers together — the output of each layer feeds into the next. It is like a pipeline: `input |> Linear |> ReLU |> Linear |> ReLU |> Linear |> output`.
- `nn.Linear(in_features, out_features)` is a fully-connected layer: it multiplies by a weight matrix and adds a bias. Like `y = W @ x + b`.
- `nn.ReLU()` is the activation function `max(0, x)`. It adds non-linearity — without it, stacking linear layers would just be one big linear layer.
- `forward` is where computation happens. PyTorch calls this when you do `model(x_t, t)`. Think of it as the `render` method.
- `torch.cat([a, b], dim=1)` concatenates tensors along dimension 1 (the feature dimension). It is like `a.map((row, i) => [...row, ...b[i]])`.

<Callout type="tip">
Why predict the noise instead of predicting the clean data directly? Both work in theory, but predicting noise leads to a simpler, more stable loss function. The network learns "what does randomness look like at this noise level?" rather than "what does the data look like?" — and that turns out to be an easier task.
</Callout>

## Training

Training a diffusion model is beautifully simple. The algorithm is just four steps repeated in a loop:

1. Sample a batch of real data points `x_0`
2. Sample random timesteps `t`
3. Sample random noise `epsilon` and create noisy data `x_t`
4. Train the network to predict `epsilon` from `x_t` and `t`

The loss is plain MSE (Mean Squared Error) between the predicted noise and the actual noise. That is it. No adversarial games, no KL divergence, no ELBO tricks. Just "guess the noise."

<CodeBlock title="train.py" language="python">
{"# Hyperparameters\nbatch_size = 256\nlearning_rate = 3e-4\nn_epochs = 5000\n\n# Create model and optimizer\nmodel = NoisePredictor(time_embed_dim=64, hidden_dim=128)\noptimizer = torch.optim.Adam(model.parameters(), lr=learning_rate)\n\n# Generate the dataset once\ndataset = make_spiral(n_points=2000)\n\nlosses = []\n\nfor epoch in range(n_epochs):\n    # 1. Sample a random batch of data points\n    idx = torch.randint(0, len(dataset), (batch_size,))\n    x_0 = dataset[idx]  # shape: (256, 2)\n    \n    # 2. Sample random timesteps for each point in the batch\n    t = torch.randint(0, T, (batch_size,))  # shape: (256,)\n    \n    # 3. Add noise to get x_t\n    x_t, noise = forward_diffusion(x_0, t, alpha_bar)\n    \n    # 4. Predict the noise and compute loss\n    noise_pred = model(x_t, t)\n    loss = nn.functional.mse_loss(noise_pred, noise)\n    \n    # Backprop and update\n    optimizer.zero_grad()  # clear old gradients\n    loss.backward()        # compute new gradients\n    optimizer.step()       # update weights\n    \n    losses.append(loss.item())\n    \n    if epoch % 500 == 0:\n        print(f'Epoch {epoch:4d} | Loss: {loss.item():.4f}')"}
</CodeBlock>

Let us walk through the training loop:

- `torch.randint(0, len(dataset), (batch_size,))` picks 256 random indices — like `Array.from({length: 256}, () => Math.floor(Math.random() * 2000))`. We use these to grab a random batch from our dataset.
- `torch.randint(0, T, (batch_size,))` picks a random timestep for each sample. This is key — each point in the batch gets noised to a different degree.
- `nn.functional.mse_loss(pred, target)` computes the mean squared error: `mean((pred - target)^2)`. This is our entire loss function.
- `optimizer.zero_grad()` clears the gradients from the last step. PyTorch accumulates gradients by default (useful in some advanced scenarios), so you have to manually zero them.
- `loss.backward()` runs backpropagation — computes how each weight contributed to the loss.
- `optimizer.step()` updates the weights using Adam (an adaptive learning rate optimizer — think of it as SGD but smarter).

**Why does predicting noise work?**

Here is the short version. Remember the forward process equation:

```
x_t = sqrt(alpha_bar_t) * x_0 + sqrt(1 - alpha_bar_t) * epsilon
```

If you know `x_t`, `t`, and `epsilon`, you can solve for `x_0`. So predicting the noise is equivalent to predicting the clean data — it is just a reparameterization. The DDPM paper showed that this noise-prediction formulation gives a loss that is equivalent to a variational lower bound on the data log-likelihood, but with lower variance. Translation: it trains more stably.

Now let us plot the training loss to make sure the model is learning:

<CodeBlock title="plot_loss.py" language="python">

## Sampling

This is the payoff. We trained the model to predict noise — now we use it to generate data from scratch.

The DDPM sampling algorithm starts from pure Gaussian noise `x_T` and iteratively denoises it, one step at a time, from t=T-1 down to t=0. At each step, the model predicts the noise in the current `x_t`, and we subtract a scaled version of that prediction to get a slightly cleaner `x_{t-1}`.

Here is the algorithm:

1. Sample `x_T` from N(0, I) — pure noise
2. For t = T-1, T-2, ..., 1, 0:
   - Predict noise: `epsilon_pred = model(x_t, t)`
   - Compute the mean of `x_{t-1}` using `epsilon_pred`
   - Add a small amount of fresh noise (except at the final step t=0)
3. Return `x_0` — your generated data

<CodeBlock title="sample.py" language="python">
{"@torch.no_grad()  # disable gradient tracking — we are not training\ndef sample(model, n_samples, T, beta, alpha, alpha_bar):\n    \"\"\"Generate samples by running the reverse diffusion process.\n    \n    Args:\n        model: trained noise predictor\n        n_samples: how many points to generate\n        T: number of timesteps\n        beta, alpha, alpha_bar: noise schedule tensors\n    \n    Returns:\n        generated points, shape (n_samples, 2)\n    \"\"\"\n    # Start from pure noise\n    x = torch.randn(n_samples, 2)\n    \n    for t_val in reversed(range(T)):\n        # Create a batch of the current timestep\n        t = torch.full((n_samples,), t_val, dtype=torch.long)\n        \n        # Predict the noise\n        eps_pred = model(x, t)\n        \n        # Compute coefficients\n        a = alpha[t_val]\n        a_bar = alpha_bar[t_val]\n        b = beta[t_val]\n        \n        # Compute the mean of x_{t-1}\n        # This is the DDPM reverse step formula:\n        # mu = (1 / sqrt(alpha_t)) * (x_t - (beta_t / sqrt(1 - alpha_bar_t)) * eps_pred)\n        mean = (1 / torch.sqrt(a)) * (x - (b / torch.sqrt(1 - a_bar)) * eps_pred)\n        \n        # Add noise (except at the final step)\n        if t_val > 0:\n            noise = torch.randn_like(x)\n            sigma = torch.sqrt(b)  # standard deviation\n            x = mean + sigma * noise\n        else:\n            x = mean  # final step: no noise added\n    \n    return x"}
</CodeBlock>

Let us break down the key formula:

```
mean = (1 / sqrt(alpha_t)) * (x_t - (beta_t / sqrt(1 - alpha_bar_t)) * eps_pred)
```

This comes from Bayes' rule applied to the forward process. In plain English: take the current noisy point, subtract the predicted noise (scaled appropriately), and rescale. The network is telling us "I think this much of what you see is noise," and we remove it.

The `if t_val > 0` check is important. During intermediate steps we add a small amount of fresh noise — this is the stochastic part that makes DDPM a probabilistic model. At the very last step we skip the noise so the output is clean.

<Callout type="info">
`@torch.no_grad()` is a decorator that tells PyTorch not to track operations for gradient computation. During sampling we are not training, so we do not need gradients. This saves memory and speeds things up significantly. It is like wrapping a section of code in a "read-only" mode.
</Callout>

{"@torch.no_grad()\ndef sample_with_history(model, n_samples, T, beta, alpha, alpha_bar, save_every=100):\n    \"\"\"Sample and save intermediate steps for visualization.\"\"\"\n    x = torch.randn(n_samples, 2)\n    history = [x.clone()]\n    \n    for t_val in reversed(range(T)):\n        t = torch.full((n_samples,), t_val, dtype=torch.long)\n        eps_pred = model(x, t)\n        \n        a = alpha[t_val]\n        a_bar = alpha_bar[t_val]\n        b = beta[t_val]\n        \n        mean = (1 / torch.sqrt(a)) * (x - (b / torch.sqrt(1 - a_bar)) * eps_pred)\n        \n        if t_val > 0:\n            x = mean + torch.sqrt(b) * torch.randn_like(x)\n        else:\n            x = mean\n        \n        if t_val % save_every == 0:\n            history.append(x.clone())\n    \n    return history\n\nhistory = sample_with_history(model, 1000, T, beta, alpha, alpha_bar, save_every=100)\n\nfig, axes = plt.subplots(1, len(history), figsize=(3 * len(history), 3))\nfor i, (ax, snap) in enumerate(zip(axes, history)):\n    ax.scatter(snap[:, 0], snap[:, 1], s=2, alpha=0.5, c='steelblue')\n    step = T - i * 100 if i > 0 else T\n    ax.set_title(f't={max(0, step)}')\n    ax.set_xlim(-3, 3)\n    ax.set_ylim(-3, 3)\n    ax.set_aspect('equal')\n    ax.grid(True, alpha=0.3)\n\nplt.tight_layout()\nplt.show()"}

That is a complete DDPM from scratch. In about 100 lines of PyTorch you built a model that generates structured data from random noise. The same principles scale up to images (just swap the MLP for a U-Net), audio, video, and 3D shapes. Stable Diffusion, DALL-E, and Midjourney all use this foundation — they just add a lot more architecture and data on top.

The key takeaways:

- **Forward process:** gradually add noise according to a schedule. No learning required.
- **Training:** predict the noise at any timestep. Simple MSE loss.
- **Sampling:** start from noise, iteratively denoise using the trained model.
- **The schedule matters.** The beta values control how fast noise is added and removed.
- **Diffusion is flexible.** The same algorithm works on any data distribution.
